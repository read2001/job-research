# 候选人经历访谈指南

你正在做一次深度访谈，目的是建立一份比简历详细得多的"经历画像"——详细到能为任何岗位改出有说服力的简历和求职材料。

## 通用原则

- **支持各种经历形态**：实习、全职工作、毕业论文、课题/合作项目、个人或开源项目都算"经历"。按对求职的重要程度排序，最重要的先聊、多聊。
- **最新、最相关的经历深聊；老旧或候选人想翻篇的轻聊，甚至跳过。**
- **全程抓具体**：含糊就追问（"大概多少？""具体几个人？""这个'提升'对应什么数字？"）。

## 访谈顺序

按"对求职最重要 → 次要"排，通常也是时间倒序。每个经历覆盖以下几块：

### 1. 背景与性质
- 这是什么经历？实习 / 全职 / 毕设 / 课题 / 个人项目？
- 做的是什么产品、解决什么问题、面向谁？
- 团队规模、候选人的角色、汇报对象或导师是谁。

### 2. 候选人做了什么
- 具体负责哪几块？设计、实现、还是其中某一部分？
- 开始时是什么状态，结束时变成什么样？
- 是自己从零搭的，还是在已有基础上改的？

### 3. 成果与指标（抓数字）
每个主要成果，按这个结构追问：
- 做了什么 / 为什么 / 怎么做 / 结果
- 结果尽量量化：性能、准确率、效率、规模、成本、用户量……
- 顺手确认下哪些是自己独立完成的、哪些是合作或借鉴现成方案的——不必纠结边界，只是避免简历写成夸大。

### 4. 技术与方法
- 用了什么工具、框架、模型、平台？
- 引入了什么方法或选型？为什么这么选。
- 评估体系是否扎实：指标定义清不清楚、有没有对照或受控实验。

### 5. 挑战与收尾
- 最难的是什么？什么没做成、学到了什么？
- 为什么离开 / 结束？留下了什么（如果有）。

## 访谈技巧

- **抓数字**："你说的'提升'大概是多少？""'很多人'是多少？""'更快'快几倍？"
- **找故事**："这段经历你最得意的是什么？""给外行 30 秒怎么讲？"
- **串主线**：留意跨经历的模式（比如总是重视评估、总是独立交付、总是把新技术落进具体场景），最后提炼出"超能力"和"叙事主线"。
- **听原话**：候选人自己的措辞是写 bullet 的金矿，记几句原话（语音输入转录可能有口误，整理时纠正）。

## 访谈后

按 SKILL.md 里画像流程给出的 profile 模板整理。尽量包含：
- 每段经历的主要工作、量化成果、技术栈
- 即便数字是大概的（"约 3 倍"）也比没有强
- 候选人描述自己工作的原话
- 跨经历的模式与超能力（用证据，不是自评）
